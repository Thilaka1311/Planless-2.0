# PLAN CARD RULES

1. Plan cards are the primary interaction surface of Planless.

2. Every plan card should feel:
- immersive
- socially alive
- touch-friendly
- lightweight
- emotionally real

3. A plan card should communicate:
“What is happening?”
within the first second.

4. The activity should always be the strongest visual element on the card.

5. Information hierarchy for every plan card must follow this structure:
- Activity
- Time
- Venue
- People attending
- Secondary details

6. The card should prioritize:
- clarity
- emotional energy
- social relevance

over:
- excessive information
- feature density
- visual complexity

7. Plan cards should use:
- full-screen or near full-screen layouts
- rounded corners
- soft dark surfaces
- layered depth
- minimal overlays

8. Cards should feel:
- swipeable
- mobile-native
- comfortable under the thumb

9. The primary image or visual should:
- support the activity
- create atmosphere
- communicate energy

without overwhelming readability.

10. Use:
- soft gradients
- subtle overlays
- restrained contrast

to preserve text readability.

11. Avoid:
- poster-heavy cinematic layouts
- excessive glow
- noisy compositions
- cluttered overlays

12. Plan cards should always prioritize:
- readability
- fast scanning
- instant understanding

13. Plan cards should not feel:
- transactional
- corporate
- enterprise-heavy
- like booking platforms

14. The card should feel:
“friends are doing something together.”

15. Metadata should remain visually secondary.

16. Avoid overcrowding cards with:
- analytics
- badges
- excessive tags
- too many buttons
- large information blocks

17. Every card should contain:
- one primary action
- one primary focus
- one clear interaction path

18. CTA buttons should remain:
- clean
- minimal
- text-first

Examples:
- Join Plan
- View Plan
- Browse

19. Avoid:
- emoji CTAs
- oversized glowing buttons
- fintech-style actions

20. Motion should remain:
- subtle
- smooth
- tactile
- responsive

21. Swipe interactions should feel:
- frictionless
- native
- fluid

22. Cards should slightly respond to touch interaction.

Examples:
- subtle scale-down on press
- lightweight motion feedback

23. Cards should support:
- spontaneous plans
- recurring plans
- scheduled events
- casual hangouts
- sports
- movies
- restaurants
- drives
- cafés

without changing the core layout system.

24. The visual system of cards should remain consistent across all categories.

25. Plan cards should avoid:
- dense text
- multi-column layouts
- dashboard widgets
- nested interactions
- hidden complexity

26. The card should feel:
- socially active
- modern
- warm
- approachable

not:
- intimidating
- overly premium
- artistically aggressive

27. Plan cards should optimize for:
- joining quickly
- understanding instantly
- emotional connection
- social momentum

28. The final emotional feeling of a Planless card should be:
“I want to be there.”
