# MEMORIES RULES

---

# Ⅰ. MEMORIES PHILOSOPHY

1. Memories exist to preserve real-world experiences created through plans.

2. Memories should feel:
- emotional
- lightweight
- personal
- socially real

3. Memories are NOT a traditional social media feed.

4. The purpose of memories is:
- capturing moments
- preserving group experiences
- reinforcing social connection
- building long-term social history

5. Memories should strengthen:
- friendships
- recurring participation
- group identity
- emotional attachment to plans

---

# Ⅱ. MEMORY CREATION RULES

6. Memories should only be created from completed plans.

7. Users should receive an “Add Memories” prompt after a plan ends.

8. Users may upload:
- photos
- videos
- lightweight reactions

9. Memory uploads should feel:
- fast
- effortless
- mobile-first

10. Avoid:
- complicated editing systems
- creator-style publishing flows
- content management complexity

11. Upload flows should prioritize:
- speed
- emotional sharing
- low friction

---

# Ⅲ. MEMORY STRUCTURE RULES

12. Every memory should remain connected to:
- the plan
- attendees
- the circle

13. Memories should appear inside:
- plan history
- circle history
- user profiles

14. Memory organization should remain:
- chronological
- lightweight
- easy to browse

15. Memories should feel tied to:
- people
- experiences
- moments

rather than standalone content.

---

# Ⅳ. MEMORY EXPERIENCE RULES

16. Memory viewing should feel:
- warm
- nostalgic
- socially alive

17. Memory UI should remain:
- immersive
- simple
- visual-first

18. Users should be able to:
- view memories
- react to memories
- share memories externally

19. External sharing should support:
- Instagram Stories
- WhatsApp
- friend groups

20. Shared memories may use:
- lightweight templates
- recap layouts
- story-style formats

21. Avoid:
- public creator feeds
- influencer systems
- algorithmic discovery
- engagement-focused mechanics

---

# Ⅴ. SOCIAL HISTORY RULES

22. Completed plans should gradually become part of:
- personal history
- circle history
- group identity

23. Memories should reinforce:
- recurring friendships
- recurring activities
- real-world consistency

24. Memories may later contribute to:
- yearly recaps
- social summaries
- “Wrapped”-style experiences

25. Wrapped-style systems should focus on:
- experiences
- friendships
- activities
- moments

not:
- vanity metrics
- popularity systems
- creator analytics

---

# Ⅵ. UI & INTERACTION RULES

26. Memory screens should prioritize:
- visuals
- people
- atmosphere
- emotional context

27. Motion should remain:
- subtle
- smooth
- emotional

28. Avoid:
- excessive animation
- distracting transitions
- content-heavy layouts

29. Memory interactions should feel:
- lightweight
- human
- socially natural

30. The emotional feeling of Memories in Planless should be:
“Remember when we did this together?”
