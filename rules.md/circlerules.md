# CIRCLE RULES

---

# Ⅰ. CIRCLE PHILOSOPHY

1. Circles are persistent social groups inside Planless.

2. Circles exist to make recurring coordination effortless.

3. A Circle should feel like:
- a real friend group
- a sports group
- a recurring social cluster
- a trusted coordination space

4. Circles should feel:
- lightweight
- socially alive
- familiar
- personal
- easy to navigate

5. The Circle experience should feel similar to:
- WhatsApp groups
- friend-group coordination

with:
- simple navigation
- familiar interaction patterns
- latest activity appearing first

6. The purpose of Circles is:
- recurring plans
- recurring people
- easier coordination
- social familiarity

7. The emotional feeling of a Circle should be:
“These are my people.”

---

# Ⅱ. CIRCLE STRUCTURE RULES

8. Every Circle should contain:
- members
- active plans
- past plans

9. Circle pages should prioritize:
- activity
- people
- coordination

10. Latest plans should always appear at the top.

11. Time indicators should remain visible and easy to scan.

12. The Circle screen should feel:
- simple
- active
- socially warm
- easy to browse

13. Circle pages should avoid:
- dashboard complexity
- dense management systems
- excessive admin tools
- enterprise community structures

14. The Circle experience should remain:
- mobile-first
- thumb-friendly
- fast to navigate

---

# Ⅲ. MEMBERSHIP RULES

15. Users should be able to:
- join circles
- be invited to circles
- participate in plans through circles

16. Circle membership should feel:
- lightweight
- trusted
- socially natural

17. Avoid:
- complicated role systems
- layered permissions
- enterprise moderation systems

18. Circles may support:
- admins
- members

but role systems should remain minimal during MVP.

---

# Ⅳ. PLAN INTEGRATION RULES

19. Circles are deeply connected to plans.

20. Circles should primarily exist as:
- recurring plan containers
- recurring social groups

21. Users should be able to:
- create plans directly inside circles
- join active circle plans quickly
- discover recurring activities

22. Active plans should always receive higher visual priority than past plans.

23. The Circle experience should encourage:
- participation
- consistency
- recurring social behavior

---

# Ⅴ. CHAT RULES

24. Chats inside Circles should remain temporary and plan-specific.

25. A chat should only open when a plan becomes active.

26. Each plan should generate its own temporary coordination chat.

27. Chats should exist primarily for:
- coordination
- updates
- logistics
- real-time interaction during plans

28. Chats should NOT become:
- endless group chats
- Discord-style communities
- permanent messaging spaces

29. Once a plan ends:
- the active chat should close
- a new chat should open for the next plan

30. Chat systems should remain lightweight during MVP.

---

# Ⅵ. CIRCLE SETTINGS RULES

31. Circle Settings should remain:
- lightweight
- simple
- easy to understand
- non-technical

32. Circle Settings should NOT feel like:
- Discord server settings
- Slack workspace settings
- enterprise admin panels

33. The settings experience should prioritize:
- quick edits
- easy management
- low cognitive load

34. Circle Settings should only include essential controls during MVP.

35. Core Circle Settings may include:
- Edit Circle Name
- Edit Circle Photo
- Edit Circle Description
- Invite Members
- Leave Circle
- Delete Circle (Admin Only)

36. Optional lightweight settings may include:
- Notification Preferences
- Plan Visibility
- Invite Permissions

37. Avoid:
- layered permission systems
- complex moderation tools
- advanced role management
- admin dashboards
- verification systems

38. Role systems should remain minimal:
- Admin
- Member

39. Circle Settings should feel similar to:
- WhatsApp group settings
- simple friend-group management

40. Settings navigation should remain:
- flat
- predictable
- easy to scan

41. Important actions should remain visually clear:
- Leave Circle
- Delete Circle
- Remove Member

42. Destructive actions should require confirmation.

43. Member management should remain:
- simple
- lightweight
- socially natural

44. Avoid:
- excessive admin controls
- hidden menus
- nested configuration systems

45. Circle Settings should prioritize:
- usability
- speed
- clarity

over:
- configurability
- advanced tooling

46. Motion and interactions should remain:
- subtle
- responsive
- functional

47. Circle Settings should visually match the rest of Planless:
- soft dark surfaces
- rounded layouts
- minimal UI
- lightweight cards

48. The final emotional feeling of Circle Settings should be:
“easy to manage without thinking too much.”

---

# Ⅶ. UI & INTERACTION RULES

49. Circle pages should prioritize:
- clarity
- participation
- fast interaction
- low cognitive load

50. Navigation inside Circles should remain:
- simple
- minimal
- predictable

51. Avoid:
- nested tabs
- complicated management systems
- excessive settings
- feature-heavy community tooling

52. Circle cards and previews should feel:
- socially active
- warm
- approachable
- lightweight

53. Motion should remain:
- subtle
- responsive
- functional

54. Avoid:
- excessive animation
- flashy transitions
- distracting interactions

---

# Ⅷ. SOCIAL BEHAVIOR RULES

55. Circles should encourage:
- recurring friendships
- recurring participation
- real-world consistency
- social momentum

56. The Circle system should optimize for:
- coordination
- participation
- recurring plans
- emotional familiarity

57. Avoid:
- influencer behavior
- creator systems
- engagement farming
- popularity mechanics

58. Circles should feel:
“real-world friend groups coordinating together.”

59. The final emotional feeling of the Circle experience should be:
“This feels like my actual social world.”
