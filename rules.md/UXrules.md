# UX RULES

---

# Ⅰ. UX PHILOSOPHY

1. Planless UX should prioritize:
- speed
- clarity
- coordination
- emotional comfort

2. Every interaction should feel:
- lightweight
- socially natural
- frictionless
- mobile-native

3. The product should optimize for:
- doing things quickly
- joining plans easily
- coordinating effortlessly

4. UX should never feel:
- corporate
- operationally heavy
- enterprise-like
- feature overloaded

5. The app should feel:
“easy to use without thinking too much.”

---

# Ⅱ. NAVIGATION RULES

6. Navigation should remain:
- simple
- predictable
- easy to learn

7. The app should prioritize:
- shallow navigation
- low tap count
- fast movement between screens

8. Avoid:
- deep nested navigation
- hidden interactions
- complex information architecture

9. Bottom navigation should remain:
- minimal
- thumb-friendly
- visually secondary

10. Users should always understand:
- where they are
- what they can do next
- how to go back

---

# Ⅲ. INTERACTION RULES

11. Interactions should feel:
- responsive
- tactile
- smooth
- native

12. Touch targets should remain:
- large
- thumb-friendly
- easy to press

13. Swipe interactions should feel:
- frictionless
- intentional
- natural

14. Motion should support:
- clarity
- feedback
- flow continuity

15. Avoid:
- over-animation
- distracting movement
- cinematic transitions
- unnecessary motion systems

16. Feedback should remain immediate.

Examples:
- loading states
- join confirmations
- payment confirmations
- success states

---

# Ⅳ. INFORMATION HIERARCHY RULES

17. Every screen should have:
- one primary focus
- one dominant action
- one clear interaction path

18. Important information should always appear first.

19. Prioritize:
- activity
- people
- timing
- venue

before secondary metadata.

20. Avoid:
- dense information clusters
- visual overload
- excessive metadata
- dashboard-style layouts

21. Screens should remain:
- visually breathable
- easy to scan
- low cognitive load

---

# Ⅴ. INPUT & FORM RULES

22. Typing should be minimized whenever possible.

23. Forms should remain:
- short
- simple
- fast to complete

24. Prefer:
- selections
- suggestions
- quick actions
- lightweight inputs

over:
- long manual entry

25. Required inputs should remain minimal during MVP.

26. Avoid:
- setup-heavy onboarding
- enterprise-style forms
- excessive configuration

---

# Ⅵ. PERFORMANCE & FLOW RULES

27. UX should optimize for:
- speed
- responsiveness
- fluidity

28. Important actions should require minimal steps.

29. Loading experiences should feel:
- lightweight
- calm
- fast

30. Avoid:
- blocking flows
- unnecessary confirmations
- friction-heavy interactions

31. The app should maintain:
- smooth scrolling
- responsive gestures
- fast transitions

---

# Ⅶ. SOCIAL UX RULES

32. Social interactions should feel:
- warm
- casual
- human
- emotionally real

33. The app should encourage:
- participation
- coordination
- recurring interaction
- real-world activity

34. Avoid:
- creator mechanics
- influencer behavior
- engagement farming systems
- vanity metrics

35. Social visibility should remain lightweight and contextual.

---

# Ⅷ. MVP UX RULES

36. Simplicity is more important than feature depth.

37. Clarity is more important than visual experimentation.

38. Fast usability is more important than advanced customization.

39. If a feature increases cognitive load significantly, simplify or remove it.

40. The final emotional feeling of Planless UX should be:
“This feels effortless and socially natural.”
